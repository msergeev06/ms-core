Описание структуры Ядра, а также строение дерева директорий пакетов (модулей)

Основная структура каталогов

www/
	msergeev/						Основной каталог. В нем хранятся все файлы ядра и дополнительных модулей
		core/						Ядро. В нем содержатся файлы, необходимые для функционирования ядра сборки
			css/					CSS файлы ядра
			entity/					Сущности
			exception/				Исключения
			images/					Основные изображения ядра
			js/						JS файлы ядра
			lib/					Классы ядра
			tables/					Таблицы DB ядра
			tools/					Вспомогательные инструменты ядра
			core.php				Главный конфигурационный файл ядра, в котором подключаются все библиотеки
			default_options.php		Опции по-умолчанию
			tools.php				Файл, в котором подключаются вспомогательные инструменты ядра из папки tools/
		packages/					Пакеты. В данной папке содержатся программные файлы модулей
			[имя_пакета]			Основной каталог пакета (модуля).
			 	css/				CSS-файлы модуля
			 	images/				Изображения модуля
			 	js/					JS-файлы модуля
				lib/				Классы модуля
				tables/				Классы с описанием таблиц DB пакета (модуля)
				templates/			Шаблоны публичной части модуля
				tools/				Инструменты модуля
				public/				Публичка. В данной папке содержатся файлы, отображаемые пользователю.
					[разделы]/		Прочие разделы публичной части модуля
					include/		Подключаемые файлы header.php и footer.php, в которых подключается нужный шаблон
					include_areas/	Подключаемый файлы
					index.php		Индексный файл публички
					include.php		Главный конфигурационный файл пакета (модуля)
			packages.php			Файл общей конфигурации пакетов. Временно не используется.
		upload/						Загружаемые пользователем файлы.
			[имя_пакета]			Каталог для загружаемых файлов модуля
		include.php					Дополнительный автоматически подключаемый файл. Служит для подключения других файлов
	index.php
	msergeev_config.php				Главный конфигурационный файл
	[прочие файлы]

Правила использования Неймспейсов

1. Каталог msergeev принимается за начало неймспейсов MSergeev
2. Далее идет название внутреннего каталога соответственно:
	core		Core
	packages	Packages
3. Далее идет название следующего каталога, в которых создаются необходимые классы.
	lib			Lib
	entity		Entity
	exception	Exception
	tables		Tables
Т.о. класс исключения ArgumentException будет располагаться в namespace MSergeev\Core\Exception



Правила наименования классов и файлов классов

Имена классов и файлов в которых они лежат совпадают. Различие в том, что в имени класса каждое слово начинается
с прописной буквы, например, ArgumentException. В имени файла же каждое слово класса пишется через подчеркивание '_',
например, argument_exception.php


Правила наименования классов таблиц, имен таблиц и их файлов

При наименовании классов таблиц каждое слово класса начинается с большой буквы а в конце добавляется слово Table,
например, ProductionCalendarTable
При наименовании таблиц DB:
	в начале идет префикс 'ms_'
	затем название модуля или core, если таблица принадлежит ядру, например, 'events_'
	затем идет название таблицы, такое же как название класса, строчными буквами, каждое слово через подчеркивание '_',
	в конце НЕ указывается '_table'.
например, ms_events_production_calendar
При наименовании файлов с описанием таблицы DB, файл называется также как и класс, без добавления 'table', каждое
слово пишется через подчеркивание '_', например, production_calendar.php
